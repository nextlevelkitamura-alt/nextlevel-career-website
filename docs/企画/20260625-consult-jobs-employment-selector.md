# 相談予約下の雇用形態別求人導線

## 目的

相談予約カレンダーで日付と相談枠を選んだ直後に、同じ文脈で確認できる求人を提示する。

ユーザーが「まず相談したい」状態でも、派遣・正社員の求人数とおすすめ求人を見せることで、予約前後の離脱を減らし、求人詳細への回遊を増やす。

## UI案

予約枠カードの直下に「求人も一緒に見る」ブロックを追加する。

- 派遣タブ: `派遣 38件` のように件数を表示
- 正社員タブ: `正社員 24件` のように件数を表示
- 選択中の雇用形態に合わせて、画像なしのおすすめ求人を2〜3件表示
- カード末尾に「すべての求人を見る」を置き、求人一覧へフィルタ付きで遷移する

既存の求人詳細ページ下部にある横スクロールのおすすめ求人と、`components/consult-jobs/ConsultationJobList.tsx` のカード表現を土台にする。

## 推奨仕様

初期表示は選択中の相談ルートに合わせる。

- 正社員相談を選んでいる場合: 正社員タブを選択
- 派遣相談を選んでいる場合: 派遣タブを選択
- 迷っている相談の場合: 件数が多い方、または派遣を初期表示

件数は「選択日だけに紐づく求人」ではなく、まずは「現在の相談ルートに合う公開求人の総数」にする方がよい。日付別に件数が変わりすぎると、カレンダー操作のたびに表示が不安定になるため。

おすすめ求人は、可能なら選択日・相談ルートに紐づく `consultation_date_jobs` を優先し、なければ公開求人から雇用形態別にフォールバックする。

求人画像は現時点では登録されていないため、カードは画像なしで組む。

- 雇用形態バッジ
- 求人タイトル
- 給与
- エリア
- 主要タグ
- `求人詳細を見る` ボタン

「すべての求人を見る」は現在の求人一覧クエリを使う。

- 正社員タブ: `/jobs?type=正社員`
- 派遣タブ: `/jobs?type=派遣`

職種カテゴリまで絞り込む場合は、既存の `category` クエリを併用して `/jobs?type=正社員&category=営業` のようにする。ただし今回の初期版は、相談ルートと一致する雇用形態フィルタだけで十分。

## 実装候補

- `components/consult-jobs/ConsultJobsClient.tsx`
  - 予約枠カードの下に新ブロックを配置
  - 選択中ルートに応じた初期タブを管理
- `components/consult-jobs/ConsultationJobList.tsx`
  - 既存カードを再利用し、見出し・件数・タブ付きの表示に拡張
- `app/consult-jobs/actions.ts`
  - 雇用形態別件数とフォールバック求人を取得する view model を追加
- `app/jobs/page.tsx`
  - 既存の `type` クエリ対応を利用するため、原則変更不要

## 受け入れ条件

- 予約枠カードの直下で、派遣・正社員の求人数を切り替え表示できる
- 選択中の雇用形態に合う画像なしのおすすめ求人が表示される
- 求人カードから `/jobs/[id]` に遷移できる
- 「すべての求人を見る」から `/jobs?type=正社員` または `/jobs?type=派遣` に遷移できる
- 相談予約ボタンの視認性を邪魔しない
- モバイルでテキストとカードが重ならない
- 紐づく求人がない場合は、空状態を出しつつ求人一覧への導線を残す
