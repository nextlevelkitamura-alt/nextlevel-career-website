---

## 📊 サマリー

### 主な発見:
1. **派遣**と**正社員**でデータ構造が大きく異なる

3. **研修・試用期間**の詳細情報が不足
4. **訴求タグの自動生成**に必要なデータが一部不足

## 🏗️ 提案: フォーム分離とDB設計

### 方針 A: 単一テーブル + 条件分岐

**メリット:**
- シンプルな構造
- 管理が容易

**デメリット:**
- NULL値が多くなる
- 派遣/正社員で不要なフィールドが混在

### 方針 B: テーブル分離（推奨）

```sql
-- 基本テーブル（共通）
jobs (
  id, title, description, requirements,
  working_hours, holidays, benefits, selection_process,
  area, type, salary, category, tags,
  nearest_station, workplace_address, workplace_access,
  attire, hair_style,
  employment_type, -- 'dispatch' or 'fulltime'
  ...
)

-- 派遣専用
dispatch_job_details (
  id (FK to jobs.id),
  client_company_name,
  training_period,
  training_salary,
  end_date,
  actual_work_hours,
  work_days_per_week,
  nail_policy,
  shift_notes,
  job_number,
  contact_person,
  contact_info,
  ...
)

-- 正社員専用
fulltime_job_details (
  id (FK to jobs.id),
  company_name,
  company_address,
  industry,
  company_size,
  established_date,
  company_overview,
  business_overview,
  probation_period,
  probation_details,
  overtime_hours,
  part_time_available,
  smoking_policy,
  annual_holidays,
  appeal_points,
  welcome_requirements,
  department_details,
  ...
)
```

**メリット:**
- それぞれの雇用形態に最適化
- NULL値が少ない
- 拡張性が高い

**デメリット:**
- JOIN が必要
- やや複雑

## 🎨 訴求ポイントの再検討

### カードUIで強調すべき項目:

#### 派遣:
1. **時給** — 最も大きく目立たせる（例: 1500円）
2. **交通費全額支給** — タグ表示
3. **最寄駅 + 徒歩X分** — アクセス重視
4. **即日スタート** — タグ表示
5. **週4〜5日** — 柔軟性
6. **服装・髪型自由** — タグ表示
7. **残業なし** — タグ表示

#### 正社員:
1. **年収** — 範囲表示（例: 300〜500万円）
2. **完全在宅** — タグ表示（該当する場合）
3. **転勤なし** — タグ表示
4. **年間休日121日** — 多さをアピール
5. **賞与年2回** — タグ表示
6. **未経験OK** — タグ表示
7. **大手グループ** — 安定性

### 自動タグ生成ロジック:

#### 給与・待遇系:
- `commute_allowance` に「全額」含む → 「交通費全額支給」
- `overtime_hours` が「なし」or「少ない」→ 「残業なし」
- `annual_holidays` >= 120 → 「年間休日120日以上」
- `salary_description` に「賞与」含む → 「賞与あり」

#### 勤務条件系:
- `start_date` に「即日」含む → 「即日スタート」
- `work_days_per_week` に「週4」含む → 「週4日OK」
- `workplace_access` に「徒歩5分以内」→ 「駅チカ」
- `type` === '正社員' && `department_details` に「転勤なし」→ 「転勤なし」

#### 働きやすさ系:
- `attire` に「自由」含む → 「服装自由」
- `hair_style` に「自由」含む → 「髪型自由」
- `nail_policy` に「OK」含む → 「ネイルOK」
- `requirements` に「未経験」含む → 「未経験OK」
