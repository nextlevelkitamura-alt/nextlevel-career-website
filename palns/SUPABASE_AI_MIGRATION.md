# Supabase Migration for AI Analysis

To enable the AI optimization features, please run the following SQL in your Supabase SQL Editor.

## 1. Add `ai_analysis` Column

This column will store the structured data generated by Gemini 3.0.

```sql
-- Add JSONB column for AI Analysis
ALTER TABLE jobs ADD COLUMN IF NOT EXISTS ai_analysis JSONB;

-- Optional: Create an index on the generated tags for faster searching
-- This assumes ai_analysis has a 'generated_tags' array
CREATE INDEX IF NOT EXISTS jobs_ai_tags_idx ON jobs USING GIN ((ai_analysis -> 'generated_tags'));
```

## 2. Verify

After running the SQL, you can verify the column exists by checking the `jobs` table schema.

## 3. Next Steps

Once this column is added, the new "AI Optimization" features in the admin panel (coming soon) will be able to save analysis results directly to the database.
