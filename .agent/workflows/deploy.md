# デプロイワークフロー

## 基本ルール
- ユーザーが「デプロイして」と明示的に指示した時のみデプロイを実行
- 一度に何度もプッシュしない
- プッシュ前に必ずローカルでビルドテスト

## 手順

### 1. ローカルでビルドテスト
```bash
npm run build
```
- エラーがあれば全て修正してから次へ
- 警告も可能な限り対応

### 2. 変更をコミット（まとめて1回）
```bash
git add .
git commit -m "変更内容の説明"
```

### 3. プッシュ（ユーザーの指示後のみ）
```bash
git push origin main
```

## 注意事項
- ESLintエラーが出たら、全て修正してから再度ビルドテスト
- 複数回のコミット・プッシュは避ける
- Vercel無料枠: 月100デプロイまで
